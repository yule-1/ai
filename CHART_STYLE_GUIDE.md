# CHART_STYLE_GUIDE

앞으로 생성하는 차트는 아래 스타일을 기본 규칙으로 한다.

## 1) 디자인 원칙
- 모던 다크 테마 기반
- 정보 전달 우선 (과한 장식 금지)
- 제목/축/범례/주석의 시각적 계층 유지

## 2) 폰트/언어
- 한글 깨짐 방지를 위해 한글 폰트를 강제 적용
- macOS 기본: `AppleGothic` 또는 `Apple SD Gothic Neo`
- `axes.unicode_minus = False` 설정으로 마이너스 기호 깨짐 방지

## 3) 기본 차트 규격
- 기본 캔버스: `12 x 6` 또는 `12 x 8`
- 해상도: `dpi 170~200`
- 배경/패널/그리드 색상 통일
- 축 눈금은 과도하지 않게, 그리드는 은은하게

## 4) 비교 차트 규칙
- 비교 시 기본값은 **Index=100** (기준일 정규화)
- 시계열 정렬 후 결측치는 기본 `dropna()` 처리
- 라벨은 종목명 + 티커 병기 권장

## 5) 이벤트 마커 규칙
- 이벤트는 세로 점선(`axvline`) + 텍스트 라벨
- 라벨은 겹침 최소화, 필요 시 축 상단 회전 표기
- 이벤트 기준일이 뉴스/키워드 기반이면 캡션에 명시

## 6) 출력물 규칙
- 기본적으로 **PNG + CSV 동시 생성**
- PNG: 시각화 결과
- CSV: 재현 가능한 원본 데이터

## 7) 저장 경로 규칙
- 루트: `ai/`
- 날짜 폴더: `ai/YYYYMMDD/`
- 필요 시 하위 폴더 사용:
  - `charts/`
  - `data/`

## 8) 파일명 규칙
- 내용이 바로 드러나게 작성
- 소문자/숫자/언더스코어 사용
- 예시:
  - `fadu_vs_marvell_index100_2y.png`
  - `xovr_1y_volume_price_with_aum.csv`

## 9) 재생성/버전 규칙
- 동일 주제 재생성 시 접미사 사용:
  - `_v2`
  - `_YYYYMMDD_HHMM`

## 10) 기본 품질 체크리스트
- [ ] 한글 폰트 정상 표시
- [ ] 제목/축/범례 가독성 확인
- [ ] 단위(USD, KRW, %) 명시
- [ ] 이벤트 라벨 출처 성격 명시
- [ ] PNG/CSV 모두 생성 확인
